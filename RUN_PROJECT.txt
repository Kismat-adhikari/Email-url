# HOW TO RUN THE EMAIL VALIDATOR PROJECT

## ============================================================================
## QUICK START (BOTH BACKEND & FRONTEND)
## ============================================================================

### Option 1: Using Windows Batch File (Easiest)
Simply double-click: `start_app.bat`
This will automatically start both backend and frontend.

## BACKEND SETUP & RUN

### Requirements
- Python 3.8 or higher
- pip (Python package manager)

### Step 1: Navigate to Project Root
```
cd c:\Users\kisma\Desktop\Email-url
```

### Step 2: Install Python Dependencies
```
pip install -r requirements.txt
```

### Step 3: Start Flask Backend Server
```
python app_anon_history.py
```

Expected Output:
```
Running on http://127.0.0.1:5000
```

**Backend runs on: http://localhost:5000**

### Environment Variables
Make sure you have a `.env` file in the root directory with:
```
FLASK_ENV=development
DATABASE_URL=your_database_url
SENDGRID_API_KEY=your_sendgrid_key (if using email features)
```

## TESTING THE APPLICATION

### Manual Testing
1. Open http://localhost:3000 in your browser
2. Navigate to "Validate" tab
3. Enter an email address
4. Click "Validate" button
5. Check results display

### Batch Validation Testing
1. Go to "Batch" tab
2. Paste multiple emails (one per line)
3. Click "Check All"
4. Verify cards display correctly with:
   - Email address
   - Valid/Invalid status
   - Details (confidence, deliverability)
   - Proper spacing and styling

### Known Issues & Fixes

**Issue: Cards look weird in Batch tab**
- Solution: CSS was updated. Clear browser cache:
  - Press Ctrl+Shift+Delete in browser
  - Clear "Cached images and files"
  - Refresh page

**Issue: Backend not connecting**
- Check if Flask is running on 5000
- Check firewall settings
- Try: http://localhost:5000/api/health

**Issue: Frontend won't load**
- Make sure npm install completed without errors
- Try deleting node_modules and reinstalling:
  ```
  rm -r node_modules
  npm install
  npm start
  ```

## AVAILABLE API ENDPOINTS

### Email Validation
- `POST /api/validate` - Validate single email
- `POST /api/batch-validate` - Validate multiple emails
- `GET /api/history` - Get validation history

### Team Features
- `GET /api/team/status` - Get team status
- `POST /api/team/create` - Create new team
- `POST /api/team/invite` - Invite team member

### User
- `GET /api/auth/me` - Get current user
- `POST /api/auth/logout` - Logout

### Health Check
- `GET /api/health` - Server status

## TROUBLESHOOTING

### Port Already in Use

**Backend (Port 5000)**
```
netstat -ano | findstr :5000
taskkill /PID <PID> /F
```

**Frontend (Port 3000)**
```
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```

### CSS Not Updating

1. Hard refresh in browser: `Ctrl+Shift+R` (Windows/Linux) or `Cmd+Shift+R` (Mac)
2. Clear browser cache completely
3. Restart development server: Kill and restart `npm start`

### Cards Still Look Weird?

The BatchResultsPaginated.css was updated on 12/22/2025. Ensure:
1. You're using the latest CSS file
2. Browser cache is cleared
3. Development server is restarted
4. No CSS errors in DevTools (F12 → Console)

### Import Errors in Frontend

1. Make sure you're in the `frontend` directory:
   ```
   cd c:\Users\kisma\Desktop\Email-url\frontend
   ```

2. Check that `node_modules` exists:
   ```
   npm install
   ```

3. For React errors, check node version:
   ```
   node --version  (should be 14+)
   npm --version   (should be 6+)
   ```

## ENVIRONMENT VARIABLES

Create `.env` file in root directory:

```

FLASK_ENV=development
FLASK_DEBUG=True
SECRET_KEY=your_secret_key_here

# Database
DATABASE_URL=postgresql://user:password@localhost/email_validator
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_supabase_key

# Email Services

# Verification Services
HUNTER_API_KEY=your_hunter_key

REACT_APP_API_URL=http://localhost:5000
```

## SUPPORT & DEBUGGING

### Enable Detailed Logging
Backend:
```python
# In app.py
app.config['DEBUG'] = True
```

Frontend:
```javascript
// In browser console
localStorage.setItem('debug', '*');
```

### Check Network Requests
1. Open DevTools (F12)
2. Go to Network tab
3. Try validation
4. Look for API calls to /api/validate
5. Check Response tab for data

### Common Responses

**Success (200)**
```json
{
  "email": "user@example.com",
  "valid": true,
  "confidence_score": 95,
  "deliverability": "High"
}
```

**Error (400)**
```json
{
  "error": "Invalid email format",
  "message": "Please provide a valid email address"
}
```
